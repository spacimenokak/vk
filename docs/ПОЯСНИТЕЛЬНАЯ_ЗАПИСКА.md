# Пояснительная записка к проекту QuizLive

## 1. Введение

QuizLive — веб-приложение для проведения интерактивных квизов в режиме реального времени. Платформа позволяет организаторам создавать опросы, настраивать правила и запускать сессии, а участникам — подключаться по коду комнаты и отвечать на вопросы синхронно.

## 2. Обоснование выбора средств разработки

### Frontend: React + Vite

React выбран как наиболее распространённый фреймворк с большим сообществом и экосистемой. Vite обеспечивает мгновенный hot-reload при разработке и быструю сборку production-версии. Альтернативы (Next.js, Vue) были рассмотрены, но для MVP без SSR Vite + React оказался оптимальным по скорости разработки.

### Backend: Node.js + Express

Единый язык JavaScript на клиенте и сервере упрощает разработку и поддержку. Express — минималистичный и проверенный фреймворк для REST API. Для MVP не требуется тяжёлый фреймворк вроде NestJS.

### Real-time: Socket.IO

Socket.IO обеспечивает надёжную доставку событий с автоматическим fallback на long-polling. Это критично для синхронного показа вопросов всем участникам. Альтернатива PartySocket рассматривалась, но Socket.IO имеет более зрелую экосистему и документацию.

### База данных: SQLite

Для прототипа SQLite не требует отдельного сервера БД, что упрощает развёртывание и демонстрацию. Схема спроектирована с учётом возможной миграции на PostgreSQL в production.

### Аутентификация: JWT

JSON Web Tokens позволяют реализовать stateless-авторизацию, что удобно при масштабировании. Пароли хешируются через bcrypt.

## 3. Этапы разработки

### Этап 1. Проектирование UI (Miro / Figma)

Спроектированы основные экраны:
- Главная страница с описанием платформы
- Регистрация / авторизация с выбором роли
- Личный кабинет (разный для организатора и участника)
- Редактор квиза с добавлением вопросов
- Комната квиза (ожидание, вопрос, лидерборд)

### Этап 2. Модель базы данных

Разработана реляционная схема из 7 таблиц:
- `users` — пользователи с ролями
- `quizzes` — квизы организаторов
- `questions` / `options` — вопросы и варианты ответов
- `quiz_sessions` — активные сессии с кодом комнаты
- `session_participants` — участники и их баллы
- `answers` — ответы на вопросы

### Этап 3. Веб-интерфейс

Реализованы 8 страниц на React с маршрутизацией (react-router-dom). Интерфейс выполнен в тёмной теме с акцентным фиолетовым цветом. Адаптивная вёрстка для мобильных устройств.

### Этап 4. Серверная часть

REST API на Express для CRUD-операций с квизами, аутентификации и управления сессиями. Middleware для проверки JWT и ролей.

### Этап 5. WebSocket

Socket.IO обработчик реализует:
- Подключение к комнате по коду
- Синхронный показ вопросов
- Приём и проверку ответов
- Обновление лидерборда в реальном времени

### Этап 6. Тестирование

Рекомендуемый сценарий тестирования:
1. Создать аккаунт организатора и участника (в разных вкладках)
2. Организатор создаёт квиз с 3+ вопросами разных типов
3. Организатор запускает сессию, участник подключается по коду
4. Проверить синхронность показа вопросов
5. Участник отвечает — проверить подсчёт баллов
6. Завершить квиз — проверить лидерборд и историю
